SQL 调优笔记

一、概述

SQL 调优是提高数据库性能的关键。通过对 SQL 语句的优化，可以大大提升数据库的响应速度，减少资源消耗，从而提升整体的应用性能。

二、SQL 调优技巧

  1. 使用索引
    • 为经常查询的列和 WHERE 子句中的条件列建立索引。
    • 避免全表扫描，尽量使用索引。
  2. **避免使用 SELECT ***
    • 只选择需要的列，避免选择不必要的数据。
  3. 使用连接（JOIN）代替子查询
    • 当可能的时候，使用连接来代替子查询，以提高效率。
  4. 优化存储引擎
    • 根据需求选择合适的存储引擎，如 InnoDB 或 MyISAM。
  5. 合理设计数据库结构
    • 规范化：减少数据冗余。
    • 反规范化：提高查询性能，但会增加数据冗余。
  6. 使用 LIMIT
    • 当只需要部分结果时，使用 LIMIT 来限制返回的行数。
  7. 避免在列上进行计算
    • 这会导致函数在每一行上都执行一次，可能导致全表扫描。
  8. 使用合适的数据类型
    • 为列选择合适的数据类型可以减少存储需求并提高查询性能。
  9. 定期进行数据库维护
  • 如优化表（OPTIMIZE TABLE），修复表（REPAIR TABLE）等。
  1. 监控和分析慢查询日志
  • 找出需要优化的慢查询，并对其进行优化。
  1. 考虑查询缓存
  • 如果数据库支持，开启查询缓存可以提高重复查询的性能。
  1. 使用 EXPLAIN 分析查询计划
  • 通过 EXPLAIN 可以查看查询的执行计划，从而找出潜在的优化点。
  1. 避免在 WHERE 子句中使用函数或运算
  • 这会导致函数在每一行上都执行一次，可能导致全表扫描。
  1. 使用预编译语句（Prepared Statements）
  • 预编译语句可以减少解析和优化时间，提高性能。
  1. 优化存储引擎和表结构
  • 根据应用需求和数据特点选择合适的存储引擎和表结构。例如，InnoDB 支持事务和行级锁定，而 MyISAM 则更适合于只读操作。
  1. 避免使用非标准的 SQL 功能
  • 使用标准 SQL 可以确保最大的兼容性。同时，也可以减少因数据库更换而导致的代码修改。
  1. 合理设置数据库参数
  • 根据硬件和工作负载调整数据库参数，如缓冲区大小、连接数等。
